Simulate the Knight's Tour

Introduction

This assignment will give you practice with Python, data structures, and design.

The Assignment

You will write a Python program that attempts to solve the Knight's Tour for a given board by brute-force methods. The Knight's Tour is an attempt to move the knight on a board, such that each space is visited exactly once. See the Web for more information.

Your program will read 3 arguments: rows, columns, attempts . The first 2 arguments are the size of the board, rows x columns. The last argument is the number of attempts to make.

The knight will start at (1, 1). Look around, see what moves are available, and randomly choose one (note, that is not the same as randomly choosing a move, and then seeing if it's legal). If you have no more moves (and you haven't covered the board), then the knight didn't make it.

You then continue with a fresh attempt, starting at (1, 1), until the knight succeeds, or you've used up the number of attempts. At the end, whichever the outcome, you will print a history of the knight's last attempt, w/a "success" or "failure" message. E.g., given a 3x3 board, output: 

FAIL:
 1  6  3 
 4  x  8 
 7  2  5

Your code

You'll need some smart way to store the moves you made on the board.

Output

The history of the last attempt, as described above.

Submission

You can get some practise writing makefiles, and referencing environment variables. When I run your stuff, I will fill in 3 variables: KNIGHT_ROWS KNIGHT_COLS KNIGHT_ATTS . Do not define these variables in your makefile. The rule that runs your code might look something like:

python yourfile $$KNIGHT_ROWS $$KNIGHT_COLS $$KNIGHT_ATTS

Note the 2 $$ . To test your code you can add the vars to your environment (don't forget to export them), or use the following form, which only affects the subshell's environment:

KNIGHT_ROWS=3 KNIGHT_COLS='8' KNIGHT_ATTS='40' make run

Or, create other test targets in your makefile that hardcode inputs for you.

Submit the following files:

  • makefile — with the following targets (at a minimum):
    • view – display all of your source code, using the pager less
    • run – runs your program, pulling the arguments from the environment, as described above.

      Make sure you turn echoing off, so make doesn't echo the command back to me.

    • clean – Removes any intermediate files (in this case, things w/a pyc extension)
  • Any source files. .pyc files will be deleted before I start, so, must be built from source.
  • README optional - anything you want to say to me before I grade.
