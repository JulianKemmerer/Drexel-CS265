Assignment - get Philly concert info

Introduction

The purpose of this assignment is for students to become familiar with writing simple Python scripts, to use built-in containers (data structures), and to see how modules can make some daunting tasks fairly simple.

You will write a Python program that will look at Camden County Parks's site, http://www.ccparks.com/events/eventsall.php, and display concert information.

The user interface will have 2 stages:

  1. List, in alphabeticl order, all the artists that are in town.
  2. Allow the user to choose (menu-style); then display the date(s) and the venue(s) (along w/the artist).

You will, after some input (you decide), bring the user back to the menu, which will also have a "quit" option.

Note, an artist may have more than one gig in the area; all of the relevant information must be stored.

What I'm thinking (for retrieval purposes) is a dictionary of lists, very much as was done in the markov.py program.

You might need to use something else to store information for the menu you present to the user.

Don't go too nuts w/the parsing. For this assignment, Chi-lites and Chilites, e.g., can be considered as 2 distinct artists.

urllib module - making an HTTP request

The urllib module lets you request a webpage and capture the page that is returned. We're going very simply here; We won't fill in agent info, check for an updated page, etc. Just open a connection, get the file down, and parse it.

I have provided an example of this in getConcertInfo.py. You can slurp the entire file down into a single string, or each line as an element in a list. Whichever you prefer for parsing (note: you'll want to look a bit more closely at regexps; as presented in earlier examples using awk or sed, a regexp didn't cross a newline. But they can, depending on what you want).

Some other notes

While you're working on the parsing bit I would simply pull down the page to local disk, and open it from there, to avoid maybe annoying the XPN webmasters. It's only polite, especially since you're not identifying yourself.

So, you can open it from Python as you would any other file. When you're ready, just change the open to go get the actual file from the Web (this returns a file-like object, has the same read methods, etc.)
